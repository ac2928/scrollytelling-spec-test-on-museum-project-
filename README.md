# Scrollytelling Test: Spec-Driven Development

## 1. The Assignment
This is a small test project separate from my final museum site. The goal was to implement a high-end "scrollytelling" experience (pinned background with sliding narrative) using the Spec-Driven Development (SDD) process.

## 2. The Process (Senior Engineer Workflow)
I followed the SDD methodology to focus the AI's intelligence:
- **Phase 1: SPEC.md** - I defined the "states" of the scroll (0%, 33%, 66%, 100%) before writing any code.
- **Phase 2: AGENTS.md** - I assigned the AI specific roles (Architect, Stylist, Interaction Engineer) to prevent code drift and ensure Swiss Minimal design coherence.
- **Phase 3: SPRINT.md** - Work was broken down into modular steps (Structure -> Styling -> JS Triggers).

## 3. Techniques Implemented
- **IntersectionObserver API:** Used to detect scroll depth and trigger visual transitions.
- **Sticky Positioning:** The "Pinned Media" technique used in professional narrative sites (like the BSEAI degree page).
- **Swiss Minimal Aesthetic:** High-contrast typography and a Sage archetype color palette to maintain visual authority.

## 4. Reflection
Using the Spec-Driven process allowed me to build a complex, scroll-linked interactive page in a fraction of the time it would take to "guess and check" with standard prompting. The separation of the specification from the implementation ensured that the final result matched the architectural vision exactly.
